## Trabajo 1 — Sitio Blade/MVC paralelo al panel

**Commit:** `2c475c6`

### Problema
Todo el sistema dependía de un panel de administración basado en un paquete
externo. Se necesitaba una interfaz web propia (controladores y vistas Blade),
más simple de mantener y sin atarse a las convenciones del paquete, empezando
por los módulos centrales: Equipos, Funcionarios, Marcaciones y Usuarios.

### Archivos modificados
| Archivo | Cambio |
|---|---|
| `app/Http/Controllers/EquipoController.php` | CRUD de equipos + probar conexión y ver marcaciones. |
| `app/Http/Controllers/MarcacionController.php` | Listado de marcaciones del SIA (solo lectura). |
| `app/Http/Controllers/PersonaController.php` | Listado y ficha de funcionarios. |
| `app/Http/Controllers/UserController.php` | CRUD de usuarios del sistema. |
| `app/Http/Requests/StoreEquipoRequest.php`, `UpdateEquipoRequest.php` | Validación de equipos. |
| `app/Http/Requests/StoreUserRequest.php`, `UpdateUserRequest.php` | Validación de usuarios. |
| `resources/views/layouts/app.blade.php` | Plantilla base del sitio (sidebar, topbar, estilos). |
| `resources/views/equipos/*`, `usuarios/*`, `funcionarios/index`, `marcaciones/index` | Vistas Blade de cada módulo. |
| `routes/web.php` | Rutas del sitio Blade. |
| `bootstrap/app.php` | Ajuste de arranque. |
| `tests/Feature/EquipoControllerTest.php`, `MarcacionControllerTest.php`, `PersonaControllerTest.php`, `UserControllerTest.php` | Pruebas de cada módulo. |

### Solución
Se construyó un sitio web completo en Blade con el patrón MVC clásico:
controladores por módulo, validación en Form Requests, vistas con una
plantilla común (sidebar + topbar) y sus rutas. Convive con el panel anterior
sobre la misma base de datos, así se puede migrar módulo por módulo sin
apagar nada. Todo el trabajo quedó cubierto con pruebas automáticas
(1329 líneas nuevas entre código y pruebas).

### Verificación
```bash
php artisan test --compact
```
