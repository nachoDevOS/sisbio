## Trabajo 1 — Módulo de Funcionarios completo + documentación técnica

**Commit:** `3c77995`

### Problema
Los funcionarios (tabla `Personas` del SIA, SQL Server remoto) solo se podían
listar. Faltaban el alta y la edición con validación acorde a la tabla
legada, la ficha con sus marcaciones y el catálogo de profesiones. Tampoco
había documentación de cómo se habla con los biométricos ni cómo conecta la
base SIA.

### Archivos modificados
| Archivo | Cambio |
|---|---|
| `app/Http/Controllers/PersonaController.php` | Alta, edición y ficha con marcaciones filtradas. |
| `app/Http/Requests/StorePersonaRequest.php`, `UpdatePersonaRequest.php` | Validación espejo de la tabla legada del SIA. |
| `app/Models/Sia/Persona.php` | Accesores (trim de `char()`), relaciones y formato de fecha ISO para SQL Server 2008. |
| `app/Models/Sia/Profesion.php`, `database/factories/Sia/ProfesionFactory.php` | Catálogo de profesiones. |
| Pantalla de Funcionarios (panel) | Paridad del módulo (formulario, ficha, páginas). |
| `resources/views/funcionarios/_form.blade.php` | Formulario de alta/edición en Blade. |
| `docs/COMUNICACION-BIOMETRICOS.md` | Documento del protocolo con los equipos y pendientes. |
| `docs/CONEXION-BD.md` | Documento de la conexión a la base SIA. |

### Solución
Se completó la gestión de funcionarios respetando las restricciones de la
tabla legada del SIA (tamaños, campos obligatorios, formato de fecha que SQL
Server 2008 interpreta bien en cualquier idioma). La ficha muestra las
marcaciones del funcionario filtradas por fecha. Se documentó por escrito
cómo se comunica el sistema con los biométricos y cómo se conecta a la base
del SIA, incluyendo lo que faltaba por implementar.

### Verificación
```bash
php artisan test --compact
```
