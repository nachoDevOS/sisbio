## Trabajo 1 — Conexión a SQL Server 2008 R2 remoto (SIA)

**Commit:** `(histórico)` — `feat: conexión sqlsrv a base SIA remota`

### Problema
El sistema necesitaba leer datos de la base `SIA_DEV` alojada en un
SQL Server 2008 R2 (10.50.1600.1) remoto en la red institucional
(`192.168.192.31:1433`). Por la antigüedad del motor, los drivers modernos
fallan en el handshake TLS si la conexión intenta cifrarse.

### Archivos modificados
| Archivo | Cambio |
|---|---|
| `config/database.php` | Nueva conexión `sia` (driver `sqlsrv`) con `encrypt=no` y `trust_server_certificate=true`. |
| `.env` | Bloque `DB_*_SIA` con host, puerto, base y credenciales (solo local, no se versiona). |
| `.env.example` | Mismo bloque sin valores sensibles, como plantilla. |

### Solución
Se verificó primero la cadena completa antes de tocar Laravel: red
(`Test-NetConnection` al puerto 1433), login con .NET SqlClient contra
`SIA_DEV`, extensión PHP `pdo_sqlsrv` (PECL, build TS vs16 x64 para
PHP 8.3.30) y el "ODBC Driver 17 for SQL Server" x64 instalado en Windows.
Luego se definió la conexión `sia` en `config/database.php` leyendo todo desde
variables de entorno, con `encrypt => 'no'` y
`trust_server_certificate => true`, obligatorios para que el driver moderno
acepte el TLS antiguo de SQL Server 2008 R2. Las credenciales viven únicamente
en `.env` (la contraseña entre comillas por sus caracteres especiales) y el
archivo está excluido por `.gitignore`.

### Verificación
```bash
php artisan tinker --execute "var_dump(DB::connection('sia')->select('SELECT TOP 3 name FROM sys.tables'));"
```
Devuelve tablas reales de `SIA_DEV` (`Facturas`, `dtproperties`, `Stock_IN`),
confirmando red, login y driver funcionando de punta a punta.

## Trabajo 2 — Visualización de la asistencia del SIA

**Commit:** `(histórico)` — `feat: módulo de asistencia SIA`

### Problema
Con la conexión lista, el sistema aún no mostraba nada de la base SIA. Además,
la paginación moderna de Laravel usa `OFFSET/FETCH`, sintaxis que SQL Server
2008 R2 no soporta: cualquier listado paginado rompía en la página 2.

### Archivos modificados
| Archivo | Cambio |
|---|---|
| `app/Database/SqlServer2008Grammar.php` | Reescribe consultas con offset usando `ROW_NUMBER()` (compatible 2008). |
| `app/Database/SqlServer2008Connection.php` | Conexión sqlsrv que usa ese grammar. |
| `app/Providers/AppServiceProvider.php` | Registra la conexión para el driver sqlsrv. |
| `app/Models/Sia/Persona.php`, `Asistencia.php` | Modelos de solo lectura sobre la conexión `sia` (+ factories). |
| Pantalla de Marcaciones | Listado con filtro por rango de fecha y tipo, búsqueda por funcionario. |
| Pantalla de Funcionarios | Listado del personal del SIA con búsqueda. |
| Tablero de inicio | Tarjetas de asistencia (hoy, personas, mes) y gráfico de marcaciones por día. |
| `tests/**` | 16 pruebas nuevas (grammar y consultas) con la conexión SIA simulada en sqlite. |

### Solución
Se creó un grammar que convierte la paginación a `ROW_NUMBER() OVER (...)`,
verificado contra el servidor real (121 páginas de marcaciones del mes). Los
modelos `Persona` y `Asistencia` son de solo lectura; la tabla `Asistencia`
tiene clave primaria compuesta, así que el listado define su propia clave de
fila. Las consultas del tablero se cachean cinco minutos y sin polling para no
castigar al servidor 2008, y si el SIA no responde el sistema sigue en pie. El
SIA arrastra marcaciones con fechas basura (años 2064/2103): el filtro por
defecto (mes actual hasta hoy) las deja fuera de listado, tarjetas y gráfico.

### Verificación
```bash
php artisan test --compact   # 28 pruebas en verde
```

## Trabajo 3 — Corrección de ordenamiento y ajustes visuales

**Commit:** `(histórico)` — `fix: orden por fecha en marcaciones y layout`

### Problema
Al ordenar por la columna «Fecha» de Marcaciones, SQL Server devolvía:
*"Se ha especificado más de una vez la misma columna en la lista ORDER BY"*.
El orden por defecto se sumaba al orden elegido por el usuario, repitiendo la
columna. Además se pidió quitar el buscador global superior y que el menú
lateral llegue hasta el borde superior de la pantalla.

### Archivos modificados
| Archivo | Cambio |
|---|---|
| Listado de Marcaciones | Orden por defecto que no repite la columna; la Hora acompaña como desempate. |
| Configuración de apariencia | Sin buscador global; menú lateral fijo desde el borde superior. |
| `routes/web.php` | La raíz `/` redirige al inicio (login si no hay sesión). |
| Marca institucional | Logo (`public/image/icon.png`) y favicon de la pestaña. |
| `tests/Feature/MarcacionResourceTest.php`, `ExampleTest.php` | Pruebas de orden y redirección de la raíz. |

### Solución
El orden por defecto se ajustó para que, cuando el usuario ordena por Fecha, no
se repita la columna en el ORDER BY, con la Hora como desempate y manteniendo
la paginación determinista. En lo visual, se quitó el buscador global y el menú
lateral pasó a ocupar toda la altura de la pantalla.

### Verificación
```bash
php artisan test --compact   # 29 pruebas en verde
```

**Responsable:** nachoDev

**Actividad:** Se conectó el sistema a la base de datos institucional SIA que
funciona en un servidor SQL Server antiguo de la red interna, y se agregaron

El sistema no podía consultar los datos de la base SIA porque el servidor usa
una versión antigua de SQL Server (2008 R2) que rechaza las conexiones

paso, y las credenciales quedaron guardadas de forma segura fuera del código.

**Detalle de lo entregado:**
1. Marcaciones: listado con filtro por fechas, tipo, búsqueda por funcionario
   y ordenamiento por fecha corregido.
2. Funcionarios: listado del personal registrado en el SIA con su PIN de reloj.
3. Tablero: tarjetas del día y del mes, y gráfico de los últimos 14 días.
4. Protección: si el servidor SIA no responde, el sistema sigue funcionando.
5. Apariencia: se quitó el buscador superior y el menú lateral ocupa toda la
   altura, con la identidad institucional.

**Resultado:** El sistema consulta la base `SIA_DEV` correctamente y muestra
las marcaciones, los funcionarios y el resumen de asistencia; 29 pruebas
automáticas en verde.
