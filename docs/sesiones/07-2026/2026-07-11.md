## Trabajo 1 — Conexión a SQL Server 2008 R2 remoto (SIA)

**Commit:** `(pendiente de commit)` — `feat: conexión sqlsrv a base SIA remota`

### Problema
El sistema necesitaba leer datos de la base `SIA_DEV` alojada en un
SQL Server 2008 R2 (10.50.1600.1) remoto en la red institucional
(`192.168.192.31:1433`). Por la antigüedad del motor, los drivers modernos
fallan en el handshake TLS si la conexión intenta cifrarse.

### Archivos modificados
| Archivo | Cambio |
|---|---|
| `config/database.php` | Nueva conexión `sia` (driver `sqlsrv`) con `encrypt=no` y `trust_server_certificate=true`. |
| `.env` | Bloque `DB_*_SIA` con host, puerto, base y credenciales (solo local, no se versiona). |
| `.env.example` | Mismo bloque sin valores sensibles, como plantilla. |

### Solución
Se verificó primero la cadena completa antes de tocar Laravel: red
(`Test-NetConnection` al puerto 1433), login con .NET SqlClient contra
`SIA_DEV`, extensión PHP `pdo_sqlsrv` (PECL, build TS vs16 x64 para
PHP 8.3.30) y el "ODBC Driver 17 for SQL Server" x64 instalado en Windows.
Luego se definió la conexión `sia` en `config/database.php` leyendo todo desde
variables de entorno, con `encrypt => 'no'` y
`trust_server_certificate => true`, obligatorios para que el driver moderno
acepte el TLS antiguo de SQL Server 2008 R2. Las credenciales viven únicamente
en `.env` (la contraseña entre comillas por sus caracteres especiales) y el
archivo está excluido por `.gitignore`.

### Verificación
```bash
# Comillas dobles por fuera: en cmd.exe el "->" con comillas simples
# se interpreta como redirección y crea un archivo vacío.
php artisan tinker --execute "var_dump(DB::connection('sia')->select('SELECT TOP 3 name FROM sys.tables'));"
```
Devuelve tablas reales de `SIA_DEV` (`Facturas`, `dtproperties`, `Stock_IN`),
confirmando red, login y driver funcionando de punta a punta.

## Trabajo 2 — Visualización de la asistencia del SIA en el panel

**Commit:** `(pendiente de commit)` — `feat: módulo de asistencia SIA en el panel`

### Problema
Con la conexión lista, el sistema aún no mostraba nada de la base SIA. Además,
la paginación moderna de Laravel usa `OFFSET/FETCH`, sintaxis que SQL Server
2008 R2 no soporta: cualquier listado paginado rompía en la página 2.

### Archivos modificados
| Archivo | Cambio |
|---|---|
| `app/Database/SqlServer2008Grammar.php` | Reescribe consultas con offset usando `ROW_NUMBER()` (compatible 2008). |
| `app/Database/SqlServer2008Connection.php` | Conexión sqlsrv que usa ese grammar. |
| `app/Providers/AppServiceProvider.php` | Registra la conexión para el driver sqlsrv. |
| `app/Models/Sia/Persona.php`, `Asistencia.php` | Modelos de solo lectura sobre la conexión `sia` (+ factories). |
| `app/Filament/Resources/Marcaciones/**` | Listado de marcaciones: filtro por rango de fecha y tipo, búsqueda por funcionario. |
| `app/Filament/Resources/Personas/**` | Listado de funcionarios del SIA con búsqueda. |
| `app/Filament/Widgets/SiaAsistenciaStats.php` | Tarjetas: marcaciones hoy, personas que marcaron, mes, funcionarios. |
| `app/Filament/Widgets/SiaMarcacionesChart.php` | Gráfico de marcaciones por día (últimos 14 días). |
| `tests/**` | 16 pruebas nuevas (grammar, recursos y widgets) con la conexión SIA simulada en sqlite. |

### Solución
Se creó un grammar que convierte la paginación a `ROW_NUMBER() OVER (...)`,
verificado contra el servidor real (121 páginas de marcaciones del mes). Los
modelos `Persona` y `Asistencia` son de solo lectura; la tabla `Asistencia`
tiene clave primaria compuesta, así que el listado define su propia clave de
fila. Las consultas del tablero se cachean cinco minutos y sin polling para no
castigar al servidor 2008, y si el SIA no responde el panel sigue en pie. El
SIA arrastra marcaciones con fechas basura (años 2064/2103): el filtro por
defecto (mes actual hasta hoy) las deja fuera de listado, tarjetas y gráfico.

### Verificación
```bash
php artisan test --compact   # 28 pruebas en verde
```
Panel: menú «Asistencia SIA» → Marcaciones / Funcionarios; tablero con
tarjetas y gráfico de los últimos 14 días.

## Trabajo 3 — Corrección de ordenamiento y ajustes visuales del panel

**Commit:** `(pendiente de commit)` — `fix: orden por fecha en marcaciones y layout del panel`

### Problema
Al hacer clic en la columna «Fecha» de Marcaciones, SQL Server devolvía:
*"Se ha especificado más de una vez la misma columna en la lista ORDER BY"*.
Filament sumaba el orden por defecto (definido como closure) al orden elegido
por el usuario, repitiendo la columna. Además se pidió quitar el buscador
global del topbar y que el sidebar llegue hasta el borde superior de la
pantalla (estilo SISCOR/AdminLTE).

### Archivos modificados
| Archivo | Cambio |
|---|---|
| `app/Filament/Resources/Marcaciones/Tables/MarcacionesTable.php` | Orden por defecto como string (Filament lo omite si el usuario ya ordena por esa columna); la Hora acompaña como desempate en `->sortable(['Fecha', 'Hora'])`. |
| `app/Providers/Filament/AdminPanelProvider.php` | `->globalSearch(false)`: sin buscador en el topbar. |
| `resources/views/filament/theme.blade.php` | Sidebar abierto fijo desde el borde superior; topbar y contenido arrancan a su derecha. |
| `routes/web.php` | La raíz `/` redirige al panel (login si no hay sesión) en vez de la vista de bienvenida de Laravel. |
| `app/Providers/Filament/AdminPanelProvider.php` | Logo institucional (`public/image/icon.png`) como marca del panel y favicon de la pestaña. |
| `tests/Feature/MarcacionResourceTest.php`, `ExampleTest.php` | Pruebas: ordenar por fecha en ambas direcciones y redirección de la raíz. |

### Solución
El orden por defecto pasó de closure a string `->defaultSort('Fecha', 'desc')`:
cuando el usuario ordena por Fecha, Filament omite el default y no se repite
la columna en el ORDER BY. El desempate por Hora quedó dentro del `sortable`
de la columna, manteniendo la paginación con `ROW_NUMBER()` determinista.
En lo visual, el buscador global se desactivó a nivel de panel y el CSS del
tema hace que el sidebar ocupe toda la altura de la pantalla, con el topbar
a su derecha.

### Verificación
```bash
php artisan test --compact   # 29 pruebas en verde
```
Panel: clic en la columna «Fecha» de Marcaciones ordena sin error en ambas
direcciones; topbar sin buscador; sidebar hasta arriba.
