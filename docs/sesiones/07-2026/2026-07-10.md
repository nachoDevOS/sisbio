## Trabajo 1 — Roles y permisos

**Commit:** `(histórico)` — `feat: roles y permisos`

### Problema
El sistema no tenía control de acceso: cualquier usuario autenticado podía
hacer todo. Se necesitaban roles y permisos administrables.

### Archivos modificados
| Archivo | Cambio |
|---|---|
| `composer.json` | Dependencia `spatie/laravel-permission`. |
| `app/Models/User.php` | Trait `HasRoles`. |
| `config/permission.php`, `database/migrations/*_create_permission_tables.php` | Tablas de permisos. |
| `app/Policies/{EquipoPolicy,RolePolicy}.php` | Políticas de autorización. |

### Solución
Se instaló el módulo de permisos (Spatie), se agregó `HasRoles` al modelo
`User` y se crearon las políticas y permisos. Se asignó el rol `super_admin`
al usuario administrador.

**Después:**
```php
// app/Models/User.php
use Spatie\Permission\Traits\HasRoles;

class User extends Authenticatable
{
    use HasFactory, HasRoles, Notifiable;
}
```

## Trabajo 2 — Recorte de permisos a 5 por recurso

**Commit:** `(histórico)` — `chore: limitar permisos a ver/crear/editar/eliminar`

### Problema
Se generaban muchos permisos por recurso (soft-delete, replicate, reorder…)
que el sistema no usa. Solo se necesitan 5: ver listado, ver, crear,
actualizar, eliminar.

### Archivos modificados
| Archivo | Cambio |
|---|---|
| Configuración de permisos | Métodos reducidos a solo 5; `super_admin` con acceso total. |
| `app/Policies/{EquipoPolicy,RolePolicy}.php` | Políticas con 5 métodos. |

### Solución
Se recortaron los permisos a `viewAny/view/create/update/delete` y se borraron
de la base los permisos sobrantes. Quedaron 10 permisos (5 × Equipo, 5 × Role).

## Trabajo 3 — Gestión de Usuarios

**Commit:** `(histórico)` — `feat: gestión de usuarios y asignación de roles`

### Problema
No había forma de crear ni administrar usuarios del sistema ni de asignarles
roles.

### Archivos modificados
| Archivo | Cambio |
|---|---|
| Pantalla de Usuarios | Registro, listado y edición de usuarios. |
| `app/Policies/UserPolicy.php` | Política con los 5 permisos. |
| `tests/Feature/UserResourceTest.php` | Pruebas de la pantalla. |
| `tests/Pest.php` | Helper `asSuperAdmin()`. |

### Solución
Pantalla de usuarios con Nombre, Correo (único), Contraseña (hasheada por el
cast del modelo, opcional en edición) y selector múltiple de Roles. Listado
con roles como etiquetas y acciones crear/editar/borrar. Redirige al listado
tras guardar, igual que Equipos.
